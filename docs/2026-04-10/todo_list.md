# Seedie App TODO List (2026-04-10)

> 汇总登录与账号体系落地后的下一步工作清单，优先保障可用性、可观测性与后续扩展空间。

## 一、 个人资料与账号入口
- [ ] **点击头像弹出编辑 Profile 界面**: 支持查看/编辑昵称、头像（占位即可）、手机号展示状态；保存后写回 Supabase（profiles / user_metadata 视字段归属决定）。
- [ ] **添加登出功能按钮**: UI 提供“退出登录”，绑定到 `AuthService.logout()`；登出后路由回登录流程并清空内存会话。

## 二、 登录与会话流程打磨
- [ ] **启动阶段状态展示优化**: 将“检查中”的圈圈替换为更符合产品气质的开屏页（或复用现有 Splash 视觉），并区分“初始化/无网/刷新失败”的提示文案。
- [ ] **离线策略明确化**: 明确离线时是否允许进入主流程；若允许，定义“离线可用页面/按钮置灰/提示语/重试入口”。
- [ ] **业务资料补齐容错**: profiles/students 拉取失败时不影响使用，但需要可重试、可提示，并避免重复请求造成抖动。
- [ ] **会话变化统一监听**: 当用户被远端撤销、Token 刷新失败或主动登出时，确保全局能立即一致地回到登录流程（避免出现 UI 假登录）。

## 三、 后端接口与性能优化（Supabase）
- [ ] **合并业务会话接口**: 用一个 RPC 一次返回 `role/agency_id/studentName/display_name` 等，替代 profiles + students 两次请求，降低启动与登录时延。
- [ ] **绑定流程状态完善**: phone 绑定后更新 profiles.status、phone_updated_at；后续支持 phone_verified 的真实校验（可选）。
- [ ] **RLS 与触发器联动回归测试**: 覆盖学生/代理/公司管理员三种角色的核心读写路径，确保 RLS、触发器与 RPC 不互相打架。

## 四、 可观测性与排障体验
- [ ] **登录链路耗时定位**: 在调试构建中记录关键阶段耗时（Initializing、Authenticated、业务资料拉取、导航完成），用于对比不同网络环境的差异。
- [ ] **网络错误提示分级**: 区分“无网/超时/权限不足/数据不存在”，避免统一提示为“登录失败”导致误判。

## 五、 数据一致性与本地缓存策略
- [ ] **定义本地缓存边界**: 明确哪些信息可以离线缓存（昵称、头像、最近学习数据），哪些必须在线拉取（权限、机构变更）。
- [ ] **Profile 本地镜像（可选）**: 若头像/昵称需要快速展示，可考虑在 Room 中做只读镜像缓存，并在登录后/恢复会话后增量刷新。
