# Seedie App TODO List (待实现与打磨功能)

> 列举 MVP 阶段仍需继续补全的业务闭环、UI 打磨及系统优化项。

## 一、 核心业务闭环补全
- [ ] **打卡与跨天重置逻辑**: `SplashScreen` 目前仅为静态按钮。需要补充点击后向数据库写入真实 `CheckInEntity` 记录的逻辑，并在全局层增加跨天检测机制（当系统时间跨日时，自动重置 `DailyTaskEntity` 的完成状态和热力图高亮）。
- [ ] **Tab 2 真实学习子页面**: 目前 Tab 2 仅为各模块的入口网格。需要为“背单词”、“语法”等核心模块建立真实的答题/学习子页面，并在学习完成后通过 `StudyRouter` 返回 `StudyResult`，触发真实的代币与词汇量奖励。
- [ ] **全局事件总线与动效落地**: `RewardEventBus` 虽已搭建，但缺少 UI 层的渲染拦截。需要在顶层 Scaffold 接入跨页面的“金币掉落”、“烟花/撒花”等全局 Lottie 或 Compose Canvas 动画。

## 二、 UI 与交互深度打磨
- [ ] **原生资产与插画替换**: 当前应用内大量使用 Material 默认 Icon (如 `Icons.Default.CheckCircle`、`Icons.AutoMirrored.Filled.MenuBook`)。需要替换为符合 Garden IP 的精美矢量插画 (VectorDrawable) 或真实贴图 (如实际的种子、植物三阶段原画、成就徽章独立设计)。
- [ ] **启动页 (Splash) 视觉强化**: 补充全屏精美插画铺底（如清晨的森林），而非目前的纯色背景，增强应用启动的仪式感。
- [ ] **天气与环境系统 (Garden)**: 在 Tab 3 的 `GardenPlotSection` 土地网格上方，增加悬浮的天气/环境层，依据用户的连续打卡频率或学习时长，动态切换“晴天”、“雨天”或“彩虹”等视觉效果。
- [ ] **中控指示器 (Slot API) 升级**: 将 `MainScreen` 的长条形圆点指示器，利用预留好的 Slot 架构，替换为符合 IP 设定的微型组件（如滑动到特定页时，圆点变为一朵含苞待放的花）。

## 三、 系统联调与性能体验优化
- [ ] **音效反馈**: 为导航点击、任务勾选完成、植物升级 (Level Up)、代币入账等关键节点，补充轻量级、不突兀的物理或自然音效（如水滴声、树叶摩擦声）。
- [ ] **空状态 (Empty States) 与引导**: 为“没有学习数据时的折线图”、“还没开始打卡时的空热力图”等情况，补充精美的占位图 (Placeholder) 和新手指引 (Onboarding 提示)。
- [ ] **数据库版本迁移 (Room Migration)**: 随着后续字段增加，当前 MVP 粗暴地修改 Schema 可能会导致崩溃。需要补充规范的 Room Migration 策略，或引入 DataStore 存储部分简单的非关系型偏好设置（如是否首次打开 App）。
